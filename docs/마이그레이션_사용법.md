# 마이그레이션 사용법

> `migrate.py` 운용 가이드. 시나리오별로 어떻게 쓰는지 정리.

---

## 핵심 개념 한 번 더

**xlsx는 view, CSV가 truth source.**

- `data/*.csv`가 진실의 원천. git으로 버전 관리.
- `workbook/output/*.xlsx`는 생성물. 친구·직원에게 배포되는 view.
- 양식 변경이든 데이터 백업이든 항상 **CSV를 거쳐서** 한다.

---

## 명령 5개

| 명령 | 방향 | 용도 |
|---|---|---|
| `python migrate.py extract` | xlsx → CSV | xlsx의 데이터를 CSV로 추출만 |
| `python migrate.py mirror` | CSV → CSV | 매출 수금완료 → 6.통장 자동 행 추가 |
| `python migrate.py rebuild` | CSV → xlsx | 빈 양식 + CSV → xlsx |
| **`python migrate.py sync`** | xlsx → CSV → CSV | **일상 운영** — extract + mirror |
| `python migrate.py refresh` | xlsx → CSV → xlsx | 양식 변경 사이클 — extract + mirror + rebuild |

### 주요 차이: sync vs refresh

- **sync** — xlsx는 **그대로 유지** (Excel이 박은 수식 캐시 보존). 일상 백업 + 미러링.
- **refresh** — xlsx를 **재생성**. `create_spreadsheets.py`가 변경된 후에만 사용. 수식 캐시는 사라지므로 다시 Excel 한 번 열고 저장 필요.

옵션:
- `--xlsx PATH` — 기본 경로(`workbook/output/SL철강_5월_운영시트.xlsx`) 외 다른 파일 지정
- `--data PATH` — 기본 디렉토리(`data/`) 외 다른 곳

### 자동 미러링 (mirror)

#### 매출 → 통장 입금

매출에서 다음 조건 충족 시 6.통장에 입금 행 자동 INSERT:
- 금액(원) > 0
- 수금완료 = "O"
- 입금통장 = (법인A / 사업자A / B계좌 / 현금 / 어음) 중 하나
- 주문ID = 채워져있음 (Excel이 한 번 열려 수식 평가된 상태)

미러링 행 형식:
- 일자 = 1.매출의 납품일자 (없으면 수금예정일 → 기록일 순)
- 사업자 = 1.매출의 사업자 (B계좌→사업자로 변환)
- 통장 = 1.매출의 입금통장
- 적요 = "{거래처} ({현장}) 매출 입금"
- 입금 = 1.매출의 금액
- 분류 = "매출입금" (B계좌면 "매출입금(B통장)")
- 매칭ID = 1.매출의 주문ID (dedup 키)

#### 매입 → 통장 출금

매입에서 다음 조건 충족 시 6.통장에 출금 행 자동 INSERT:
- 합계(원) > 0 (부가세 포함)
- 결제완료 = "O"
- 출금통장 = 채워져있음
- 매입ID = 채워져있음

미러링 행 형식:
- 일자 = 2.매입의 입고일자 (없으면 기록일)
- 사업자 = 2.매입의 사업자 (B계좌→사업자로 변환)
- 통장 = 2.매입의 출금통장
- 적요 = "{매입처} ({품목}) 매입 출금"
- 출금 = 2.매입의 합계 (부가세 포함 — 실제 결제 금액)
- 분류 = "매입출금" (B계좌면 "매입출금(B통장)")
- 매칭ID = 2.매입의 매입ID (dedup 키)

#### 공통 정책

**Idempotent**: 매칭ID 기준 dedup이므로 sync 여러 번 돌려도 중복 행 없음. 사용자가 6.통장 행 수동 편집해도 보존됨 (mirror는 INSERT만, UPDATE/DELETE 안 함).

**Dedup 분류 그룹 분리**: 매출 매칭ID와 매입 매칭ID는 같은 날짜에 NNN이 충돌 가능 (양쪽 모두 YYYYMMDD-NNN 형식). 이를 막기 위해 dedup 검사 시 분류 그룹 내에서만 검사:
- 매출 미러링 dedup: 통장 분류가 `매출입금` 또는 `매출입금(B통장)` 인 행만 체크
- 매입 미러링 dedup: 통장 분류가 `매입출금` 또는 `매입출금(B통장)` 인 행만 체크

→ 매출 ID `20260507-001`과 매입 ID `20260507-001`이 같은 날 동시 존재해도 양쪽 다 통장에 미러링됨.

**경계 조건**: 통장 컬럼(매출 입금통장 / 매입 출금통장) 비어있으면 skip + 경고 출력. 사용자가 채워야 다음 sync에 미러링됨.

**사업자 정규화**: 1.매출/2.매입에서 사업자=B계좌로 입력된 행 → 6.통장 사업자 컬럼은 '사업자'로 변환 (B계좌는 사업자 산하 히든 통장, entity 차원 ≠ 통장 차원).

## 시나리오별 사용법

### 시나리오 1: 일상 운영 (xlsx 편집 후 백업)

운영 중 입력된 데이터를 CSV로 빼서 git에 commit + 매출→통장 미러링.

```bash
make sync                                       # extract + mirror
git add data/
git commit -m "data: 5월 둘째 주 동기화 (5/4~5/10)"
git push
```

**xlsx는 그대로 유지** (Excel이 박은 수식 캐시 보존). 매출 수금완료 행은 자동으로 6.통장에 입금 행으로 추가됨.

**이걸 매주 안 하면**: 양식 변경할 때 CSV가 오래된 버전이라 친구·직원이 입력한 데이터가 손실됨. 또한 매출 → 통장 미러링도 누적됨.

---

### 시나리오 2: 마이너 변경 (드롭다운 옵션 추가)

예: `4.영수증` 분류에 "교통비" 추가.

```bash
# 1. 현재 데이터 백업 (안전망)
python migrate.py extract

# 2. workbook/create_spreadsheets.py 수정
#    _build_sheet_receipts() 안의 dv_cat_r formula1에 "교통비" 추가
#    예: '"식대,차량유지,교통비,판촉물,..."'

# 3. 빈 양식 + 데이터 → 새 xlsx
python migrate.py rebuild

# 4. 변경 이력 갱신
echo "교통비 분류 추가" >> docs/워크북_변경이력.md  # 수동 편집

# 5. commit
git add -A
git commit -m "feat(영수증): 분류에 교통비 추가"
```

---

### 시나리오 3: 컬럼 추가 (호환 변경)

예: `2.매입` 시트에 "어음만기일" 컬럼 V열 추가.

```bash
# 1. 데이터 추출
python migrate.py extract

# 2. workbook/create_spreadsheets.py 수정
#    _build_sheet_purchases()의 headers에 '어음만기일' 추가
#    set_widths()에 너비 추가
#    필요 시 data validation, 조건부 서식

# 3. migrate.py의 SHEETS['2.매입']['max_col']을 21 → 22로 변경
#    날짜 컬럼이면 date_cols에 22 추가

# 4. rebuild
python migrate.py rebuild

# 5. 검증: V열이 새로 생겼고, A~U열 데이터는 그대로인지
python3 -c "
from openpyxl import load_workbook
wb = load_workbook('workbook/output/SL철강_5월_운영시트.xlsx')
ws = wb['2.매입']
print('V1 헤더:', ws['V1'].value)
print('A2 기존 데이터:', ws['A2'].value)
"

# 6. 이력 + commit
git add -A
git commit -m "feat(매입): 어음만기일 컬럼 추가"
```

### 시나리오 4: 컬럼 이름 변경 (브레이킹)

예: `3.재고`의 "안전재고" → "최소재고"로 이름 변경.

```bash
# 1. 추출
python migrate.py extract

# 2. workbook/create_spreadsheets.py 수정 (헤더만 변경, 컬럼 위치는 유지)

# 3. data/3.재고.csv 헤더도 같이 수정
#    sed -i '' '1s/안전재고/최소재고/' data/3.재고.csv  (Mac)

# 4. rebuild
python migrate.py rebuild

# 5. 검증 + commit
git commit -am "refactor(재고): 안전재고 → 최소재고 (용어 통일)"
```

**주의**: 컬럼 이름이 다른 시트의 SUMIFS/COUNTIF 수식에서 참조되면 같이 수정 필요. `현황판`, `5.거래처`, `8.미수관리` 시트의 수식을 grep으로 확인:

```bash
grep -n "안전재고" workbook/create_spreadsheets.py
```

### 시나리오 5: 컬럼 삭제 또는 위치 이동

예: `1.매출` Q열 "사진전송" 컬럼 삭제 (실효성 없어서).

```bash
# 1. 추출
python migrate.py extract

# 2. CSV에서 해당 컬럼 제거
python3 -c "
import csv
with open('data/1.매출.csv', encoding='utf-8-sig') as f:
    rows = list(csv.reader(f))
# Q열은 인덱스 16 (0-based)
new_rows = [[v for i, v in enumerate(row) if i != 16] for row in rows]
with open('data/1.매출.csv', 'w', encoding='utf-8-sig', newline='') as f:
    csv.writer(f).writerows(new_rows)
"

# 3. workbook/create_spreadsheets.py에서 Q열 관련 코드 모두 제거
#    - headers에서 '사진전송' 제거
#    - set_widths 인덱스 조정
#    - data validation Q2:Q102 제거
#    - 조건부 서식 Q열 제거

# 4. migrate.py SHEETS['1.매출']:
#    - max_col: 21 → 20
#    - formula_cols 수정 (R→Q로 시프트되면 인덱스 -1)

# 5. rebuild
python migrate.py rebuild

# 6. 매출 외 시트의 수식이 Q열을 참조하지 않는지 확인
grep -n "1.매출.*!Q" workbook/create_spreadsheets.py
```

## 트러블슈팅

### `extract` 후 CSV 파일이 비어있다

**원인**: xlsx의 헤더 행 위치가 `migrate.py`의 `SHEETS[name]['header_row']`와 다름.

**확인**:
```python
from openpyxl import load_workbook
wb = load_workbook('workbook/output/SL철강_5월_운영시트.xlsx')
ws = wb['1.매출']
for r in range(1, 5):
    print(f'{r}행 A열: {ws.cell(row=r, column=1).value}')
```

`기록일`이 나오는 행이 헤더 행. `migrate.py`의 메타 갱신.

---

### 양식의 합계 행이 데이터로 추출됐다

**원인**: 시트 양식에 `합계`/`소계` 같은 라벨 행이 데이터 영역 안에 있고, extract가 1열에 텍스트가 있다는 이유로 데이터 행으로 간주.

**해결**: `migrate.py`의 SHEETS 메타에 `skip_label_rows` 추가.

```python
'8.미수관리': {
    ...,
    'skip_label_rows': {'합계'},  # 1열이 이 값인 행은 추출 제외
},
```

다른 시트에서도 비슷한 라벨 행이 생기면 같은 방식으로 추가.

---

### `rebuild` 후 일부 행이 누락됐다

**원인**: 첫 번째 컬럼이 빈 행을 "데이터 끝"으로 간주. 첫 컬럼이 없는 정상 데이터가 있다면 룰 변경 필요.

**해결**: `migrate.py extract()` 함수의 빈 행 판단 로직 수정. 시트별로 "기준 컬럼"을 다르게 둘 수도 있음.

---

### `rebuild` 후 날짜가 텍스트로 보인다

**원인**: `migrate.py`의 `SHEETS[name]['date_cols']`에 해당 컬럼이 빠져 있음.

**해결**: 메타 갱신 + 다시 rebuild.

---

### 자동수식이 사라졌다

**원인**: `migrate.py`의 `SHEETS[name]['formula_cols']`에 해당 컬럼이 빠져 있어서 CSV 데이터로 덮어쓴 것.

**해결**: `formula_cols`에 추가하고, `create_spreadsheets.py`로 빈 양식 다시 만들어 수식 복원 후 `rebuild`.

---

### CSV가 한글이 깨진다

**원인**: BOM 없이 저장된 CSV를 Excel이 못 읽음.

**해결**: 항상 `utf-8-sig`로 read/write. `migrate.py`는 이미 그렇게 되어있음.

## 비상 복구

마이그레이션이 망가져서 워크북이 깨졌을 때:

```bash
# 1. 가장 최근 commit의 CSV로 되돌리기
git checkout HEAD -- data/

# 2. 빈 양식 + 복구된 CSV → 워크북 재생성
python migrate.py rebuild

# 3. 그래도 안 되면 git log에서 며칠 전 commit으로 갈 수도
git log --oneline data/
git checkout <commit-hash> -- data/
python migrate.py rebuild
```

CSV가 git에 있으니 어떤 시점으로든 복구 가능.

## 매일/매주 운영 룰

| 빈도 | 작업 | 명령 |
|---|---|---|
| 매주 일요일 저녁 | 데이터 백업 | `make sync && git commit` |
| 양식 변경할 때 | 변경 사이클 | `make sync` (또는 시나리오 참고) |
| 매월 말일 | 월간 백업 | `make backup` (`backup/YYYY-MM-DD/`로 복사) |
| 5월 25일 | 변경 동결 시작 | (변경 X) |
| 5월 31일 | 6월 이전 준비 | 마지막 sync + 백업 |
