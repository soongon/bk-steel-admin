## 2. 발견된 이슈와 해결

### 2-1. 5.거래처 SUMIFS 매입 누락 (commit `47ffda9`)

**문제**:
- 5.거래처 J·K 수식이 `IF(A="매출", SUMIFS(매출), "")` 형태라 매입처(OO철강, 경기메탈)는 누적 집계 안 됨.
- 또 무자료 거래처(`김씨건축` 매출, `개인 김씨` 매입)가 5.거래처 마스터에 미등록 → CLAUDE.md 절대 룰 #3(거래처명 정합성) 위반.

**해결**:
- 수식을 `IF(A="매출", SUMIFS(매출), IF(A="매입", SUMIFS(매입), ""))` 중첩 분기로 변경.
- 무자료 거래처 2건을 5.거래처 샘플 + CSV에 등록.

### 2-2. 사업자 차원 3분할 (commit `978ec37`)

**문제**:
- 매출/매입의 K열 "사업자"가 (법인/사업자) 2개뿐. 무자료 거래는 R열(입금통장=사업자B)로만 식별돼 SUMIFS·신고 분리에 누락 위험.

**해결**:
- 1.매출 K, 2.매입 K, 5.거래처 I 드롭다운에 `B계좌` 추가 → (법인/사업자/B계좌) 3분할.
- 현황판 매출/매입 영역에 B8/E8 "B계좌 (히든)" 행 추가. 기존 "합계" → "신고 합계"로 라벨 명확화.
- 영업이익(B10=B7-E7)은 신고 기준 그대로 유지 (B계좌 제외).
- **3.재고 D, 6.통장 B는 미변경** — B계좌는 통장 차원이지 entity가 아니라 재고/통장 컨텍스트엔 부적합.

### 2-4. B계좌 조건부 서식 흰 글자 가독성 (commit `eca3452`)

**문제**:
- B계좌 셀에 짙은 회색(#4A4A4A) 배경 + 흰 글자 조건부 서식 적용. 그러나 셀의 `INPUT_FONT`(파란색 #0000FF)가 명시적으로 설정돼 있어 조건부 서식의 font 규칙이 묵살됨 → 짙은 회색 위 파란 글자 = 거의 안 보임.

**해결**:
- 모든 B계좌 조건부 서식을 연한 회색(#E8E8E8) 배경만 적용, font 규칙 제거.
- 셀 INPUT_FONT(파랑) 그대로 유지 → 가독성 확보 + 시각적 구분 유지.
- 적용 위치: 1.매출 R, 2.매입 R, 6.통장 A:J, 4.영수증 F.

## 4. 운영 워크플로우 (학습 내용)

### 데이터 흐름

```
create_spreadsheets.py  +  data/*.csv
        |                       |
        └─────── rebuild ───────┘
                    ↓
         workbook/output/*.xlsx
```

- `create_spreadsheets.py` = 양식 정의 (사람이 수정)
- `data/*.csv` = 데이터 (truth source)
- xlsx = 빌드 산출물 (rebuild 시 매번 재생성)

### 명령어

| 명령 | 방향 | 용도 |
|---|---|---|
| `make extract` | xlsx → CSV | xlsx에서 편집한 데이터를 CSV로 저장 |
| `make rebuild` | CSV → xlsx | 양식 변경 후 xlsx 재생성 (자동수식 컬럼은 보존) |
| `make sync` | extract + rebuild | 양식 변경 사이클 한 번에 |
| `make build` | (샘플 포함 새 빌드) | 첫 시작 또는 fresh build |

### 안전 규칙

1. **xlsx와 CSV 동시 편집 금지** — 어느 쪽이 truth인지 모호해짐.
2. **xlsx 편집 후 extract 안 하고 rebuild 돌리면** — xlsx 편집이 CSV로 덮어씌워져 사라짐.
3. **CSV 직접 편집 후 extract 돌리면** — xlsx에는 그 데이터가 없으니 CSV에서도 사라짐.
4. **extract 전 xlsx 닫기 필수** — Excel 락이 걸리면 openpyxl이 못 읽음.
