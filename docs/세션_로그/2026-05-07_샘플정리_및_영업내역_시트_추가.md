## 11. 수식 캐시 vs 평가된 값

**현상**: rebuild 직후 1.매출의 주문ID/상태 컬럼이 비어있는 것처럼 보임 (사용자 보고).

**원인**: openpyxl이 만든 xlsx에는 수식 캐시값이 없음. Excel/Numbers에서 한 번 열어야 자동 평가되어 캐시 박힘. 미리보기/raw view에서는 비어보일 수 있음.

**수식 자체는 정상**:
- 주문ID: `=IF($A2="","",TEXT($A2,"YYYYMMDD")&"-"&TEXT(COUNTIF($A$2:$A2,$A2),"000"))`
- 상태: `=IF($D="","",IF($P="O","수금완료",IF(OR($N="O",$Q="O",$S="B계좌"),IF(AND($O<>"",$O<TODAY()),"연체","납품완료"),"주문")))`

**해결**: 사용자가 Excel에서 열고 저장 → 수식 자동 평가 → 캐시 박힘. 다음 sync로 캐시값 CSV에 영구화 (`04c5753`).

## 19. 6.통장 G열 분기 수식 (commit `3262df4`)

기존: G열(누적잔고)은 항상 row 2부터 현재 row까지 같은 통장 누적 (running). 시작잔고 행(2~4)에서는 자기 행만 누적되어 0/입금-출금만 표시 → 통장별 현재 잔고 한눈 표시 못 함.

**변경**: 분류로 분기:
```
=IF(C{row}="","",
    IF(H{row}="시작",
        SUMIFS(E:E,C:C,C{row}) - SUMIFS(F:F,C:C,C{row}),    -- 통장 전체 합계
        SUMIFS($E$2:E{row},$C$2:C{row},C{row}) - SUMIFS($F$2:F{row},...)  -- running
    ))
```

**효과**:
- G2/G3/G4 (시작잔고 행) = 그 통장의 현재 잔고
- 거래 행 G5+ = 시간 순 누적 (변경 없음)

사용자가 시작잔고 입금(E)에 실제 시작 금액 입력 → 자동 정확.

`docs/도메인_룰.md §8` 갱신.
