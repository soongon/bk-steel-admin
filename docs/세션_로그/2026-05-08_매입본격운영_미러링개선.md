## 6. 엠에스스틸 매입 3건 + 매칭ID 충돌 발견 (commits `b16de85`, `9e39d3d`)

| 20260507-003 | 13,370Kg | 11,163,950 | 1,116,395 | 12,280,345 |

### 6-2. ⚠️ 매칭ID 충돌 발견

매출/매입 모두 YYYYMMDD-NNN 형식. 5/7에:
- 매출 NNN: 001 (박영국), 002 (태웅)
- 매입 NNN: 001, 002, 003 (엠에스스틸)

mirror dedup이 매칭ID만 검사 → 매출 미러링 후 매입 미러링 시 NNN 동일하면 skip.

**누락된 매입 미러링 2건**:
- 엠에스스틸 #1 (매입ID 20260507-001) — 박영국 매출 ID와 충돌
- 엠에스스틸 #2 (매입ID 20260507-002) — 태웅 매출 ID와 충돌
- 통장에 36,399,825원 매입 출금 누락

데이터 무결성 critical 이슈.

### 6-3. Fix — 분류 그룹별 dedup

```python
SALES_CATEGORIES = {'매출입금', '매출입금(B통장)'}
PURCHASE_CATEGORIES = {'매입출금', '매입출금(B통장)'}

# 매출 mirror dedup: 통장 분류가 SALES_CATEGORIES인 행만 체크
# 매입 mirror dedup: 통장 분류가 PURCHASE_CATEGORIES인 행만 체크
```

같은 매칭ID라도 분류 그룹이 다르면 양쪽 다 미러링 가능. 5/7 매출 001 + 매입 001 양쪽 다 통장에 들어감.

검증: 누락된 매입 #1 (24M), #2 (12M) 자동 미러링 추가 확인.

`docs/마이그레이션_사용법.md` 자동 미러링 섹션에 dedup 룰 명시.

### 7-2. 부가세 수식 분기

```
변경 전: =IF(M="","",ROUND(M*0.1,0))
```

룰: 사업자(Q) **또는** 출금통장(X) 중 하나라도 'B계좌'면 부가세 0.

## 11. 학습 사항

### 매칭ID 충돌 (9.영업내역 시트도 동일 잠재 이슈)
워크북 ID 형식이 모두 YYYYMMDD-NNN. 매출/매입에 분류 그룹 분리 적용했지만, 9.영업내역의 활동ID도 같은 패턴. 영업내역과 매출/매입 사이엔 통장 매칭이 없으니 (매출 ID가 영업내역 매출ID 컬럼에 참조될 뿐) 충돌 영향 없음. 6월 시스템에서는 UUID FK라 무관.

### INSERT-only 미러링의 한계
- 사용자가 매출/매입 행의 합계를 변경하면 통장 미러링 행은 갱신 안 됨
- 이번엔 부가세 X 룰로 합계가 옛 값으로 복귀 → 우연히 정합 회복
- 일반적으로는 합계 변경 → 통장 행 수동 정리 필요
- 6월 시스템: trigger로 UPDATE 가능 (sales_id/purchase_id FK 활용)

### 도메인 룰: B계좌 부가세 처리
- B계좌(무자료) → 부가세 신고 X
- 1.매출: 사업자=B계좌 행은 현황판 SUMIFS에서 분리 표시 (이미 동작)
- 2.매입: N열 부가세 수식이 사업자 OR 출금통장 = B계좌면 0 (오늘 추가)
- 양쪽 모두 자동 분리 — 부가세 신고 시 자료 분리 자동

## 17. 사용자 워크플로우 검증 — 캐시값 운영

### 17-1. 사용자 질문

"make sync 하니까 csv 데이터 사라진다"

### 17-2. 진단

데이터 사라진 게 아니라 **수식 결과 캐시**만 비워짐:
- 사라진 것: 주문ID/상태/부가세/합계/미수금/미수일수
- 보존된 것: 거래처/현장/공급가/통장/사업자 (입력 데이터)

### 17-3. 원인 + 해결

**원인**:
- rebuild는 xlsx에 수식만 박음 (캐시 X)
- sync extract는 data_only=True로 캐시값만 읽음
- 캐시 None → csv 빈칸

**해결**:
- xlsx 한 번 열고 저장 → 자동 재계산으로 캐시 박힘
- sync → 캐시값 csv 영구화

### 17-4. 워크플로우 가이드 (사용자에게 명확화)

**일상 운영** (xlsx 편집):
```
사용자 입력 → Excel 자동 재계산 → 저장
  ↓
make sync → 캐시값 csv 반영 ✅
```

**양식 변경 후**:
```
make refresh → 새 xlsx (수식만, 캐시 X)
  ↓
사용자가 xlsx 한 번 열고 저장 → 캐시 박힘
  ↓
make sync → 캐시값 csv 반영 ✅
```

**csv 직접 편집 후**:
```
csv 편집
  ↓
make rebuild만 (extract X — sync 금지!) → xlsx 반영
  ↓
xlsx 열고 저장 → 캐시 박힘
  ↓
make sync → 캐시값 csv 영구화
```
