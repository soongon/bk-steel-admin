## 2. 사업상 중요한 포인트 — 번들 ≠ 2톤

> ⚠️ 거래처가 "2톤 주세요" 했을 때 1번들/2번들 분쟁이 흔히 발생.
> 시스템에서 가닥/번들/kg/톤 단위가 항상 명확히 환산되어야 함.

**시장 통용 번들 단위 (참고 — 제강사별 확인 필요)**

| 호칭 | 1번들 가닥 수 | 1번들 실제 중량 (8m 기준) |
|------|------------|----------------------|
| D10  | 420 가닥   | 1,881.6 kg           |
| D13  | 250 가닥   | 1,990.0 kg           |
| D16  | 150 가닥   | 1,872.0 kg           |
| D19  | 100 가닥   | 1,800.0 kg           |
| D22  |  80 가닥   | 1,945.6 kg           |
| D25  |  60 가닥   | 1,910.4 kg           |
| D29  |  50 가닥   | 2,016.0 kg           |
| D32  |  40 가닥   | 1,993.6 kg           |
| D35  |  30 가닥   | 1,802.4 kg           |
| D38  |  25 가닥   | 1,790.0 kg           |

핵심 룰:
- 일반 시판 표준 길이는 **8M** (주문생산 시 8M 외도 가능)
- 한 묶음(번들) 중량은 1,800~2,000kg 사이 — 호칭별 다름, 절대 정확한 "2톤" 아님
- 거래 시 "톤 단위" 주문은 시스템이 가닥 수 → 실제 kg로 변환해서 거래명세서에 표기
- 제강사별로 번들 단위가 약간 다를 수 있음 (현대제철·동국·한국제강 등) — 친구가 주거래처별 확인 필요

## 4. 자동 계산 함수 (트레이딩 룰)

```sql
-- =============================================
-- 가닥 수 → kg 환산 (재고 입력 시)
-- =============================================
CREATE OR REPLACE FUNCTION rebar_weight_kg(
  p_spec TEXT, 
  p_bars INT, 
  p_length_m NUMERIC DEFAULT NULL
) RETURNS NUMERIC AS $$
  SELECT unit_weight_kg_per_m * COALESCE(p_length_m, standard_length_m) * p_bars
  FROM rebar_specs WHERE spec_code = p_spec;
$$ LANGUAGE SQL IMMUTABLE;

-- =============================================
-- 톤 주문 → 필요 가닥 수 (거래명세서 자동 생성)
-- =============================================
CREATE OR REPLACE FUNCTION rebar_bars_for_tons(
  p_spec TEXT, 
  p_tons NUMERIC, 
  p_length_m NUMERIC DEFAULT NULL
) RETURNS INT AS $$
  SELECT CEIL((p_tons * 1000) / (unit_weight_kg_per_m * COALESCE(p_length_m, standard_length_m)))::INT
  FROM rebar_specs WHERE spec_code = p_spec;
$$ LANGUAGE SQL IMMUTABLE;

-- =============================================
-- 번들 수 → 실제 중량 kg (매입 발주 시)
-- =============================================
CREATE OR REPLACE FUNCTION rebar_weight_by_bundles(
  p_spec TEXT, 
  p_bundles INT
) RETURNS NUMERIC AS $$
  SELECT bundle_weight_kg * p_bundles
  FROM rebar_specs WHERE spec_code = p_spec;
$$ LANGUAGE SQL IMMUTABLE;

-- =============================================
-- 사용 예시
-- =============================================
-- SELECT rebar_weight_kg('D13', 100);              -- 796.0 (D13 100가닥 8m)
-- SELECT rebar_bars_for_tons('D13', 5);            -- 628   (D13 5톤 = 628가닥 8m, 실제 4,999.7kg)
-- SELECT rebar_weight_by_bundles('D16', 2);        -- 3744.0 (D16 2번들)
```

## 5. TypeScript 헬퍼 (`packages/types/rebar.ts`)

```ts
export interface RebarSpec {
  specCode: string;
  nominalDiameterMm: number;
  nominalAreaMm2: number;
  unitWeightKgPerM: number;
  standardLengthM: number;
  barsPerBundle: number | null;
  bundleWeightKg: number | null;
}

// 가닥 수 → kg
export function rebarWeightKg(
  spec: RebarSpec, 
  bars: number, 
  lengthM?: number
): number {
  return spec.unitWeightKgPerM * (lengthM ?? spec.standardLengthM) * bars;
}

// 톤 → 필요 가닥 수
export function rebarBarsForTons(
  spec: RebarSpec, 
  tons: number, 
  lengthM?: number
): number {
  const length = lengthM ?? spec.standardLengthM;
  return Math.ceil((tons * 1000) / (spec.unitWeightKgPerM * length));
}

// 번들 수 → 실제 kg
export function rebarWeightByBundles(spec: RebarSpec, bundles: number): number {
  if (spec.bundleWeightKg === null) {
    throw new Error(`${spec.specCode}는 번들 단위 미정 (제강사 확인 필요)`);
  }
  return spec.bundleWeightKg * bundles;
}

// 표시용 포맷
export function formatRebarOrder(
  spec: RebarSpec, 
  bars: number, 
  lengthM?: number
): string {
  const length = lengthM ?? spec.standardLengthM;
  const kg = rebarWeightKg(spec, bars, lengthM);
  const tons = (kg / 1000).toFixed(2);
  return `${spec.specCode} / ${length}M / ${bars}가닥 / ${kg.toFixed(1)}kg (${tons}톤)`;
}
```

## 6. UX 패턴 — 매출/매입 폼

### 입력 모드 토글 (3가지 단위 자유 전환)

거래처마다 주문 단위가 다름. 한 폼에서 3가지 모드 지원:

```
┌─────────────────────────────────────────┐
│ 호칭: [D13 ▼]   강종: [SD400 ▼]          │
│ 길이: [8M ▼]                             │
│                                          │
│ 입력 방식: ( ) 가닥  ( ) kg  (●) 톤      │
│                                          │
│ 톤 입력: [   5.0   ] 톤                   │
│                                          │
│ ─── 자동 환산 ───                         │
│ 필요 가닥 수: 628 가닥                    │
│ 실제 중량: 4,998.9 kg                    │
│ 거래명세서 표기: D13 / 8M / 628가닥       │
│                / 4,999kg / @단가 / 합계   │
└─────────────────────────────────────────┘
```

### 번들 단위 발주 (매입 폼)

매입은 보통 번들 단위:
```
호칭: [D16] / 번들 수: [2] → 실제 중량: 3,744kg (300가닥)
```

## 7. 시스템 활용 시나리오

| 화면 | 사용 함수 | 동작 |
|------|---------|------|
| 매출 입력 (톤 주문) | `rebar_bars_for_tons` | 5톤 입력 → 가닥 수 자동 계산 → 실제 kg 표시 |
| 매출 입력 (가닥 주문) | `rebar_weight_kg` | 100가닥 입력 → kg·톤 자동 환산 |
| 매입 입력 (번들) | `rebar_weight_by_bundles` | 2번들 → 실제 중량 자동 |
| 재고 차감 | `rebar_weight_kg` | 출고 가닥 수 → 재고 kg 차감 |
| 거래명세서 출력 | `formatRebarOrder` | 표준 포맷으로 자동 출력 |
| 견적서 자동 작성 | 위 모두 | 견적 1분 내 발송 |

## 8. 향후 확장 — 강관·각파이프

같은 패턴으로 강관·각파이프도 마스터 시드:

```sql
CREATE TABLE pipe_specs (
  spec_code TEXT PRIMARY KEY,           -- '50×50×2.0t' 형식
  outer_diameter_mm NUMERIC,            -- 사각: 한 변, 원형: 외경
  thickness_mm NUMERIC,
  shape TEXT,                           -- 'square' | 'rectangular' | 'round'
  unit_weight_kg_per_m NUMERIC,
  standard_length_m INT DEFAULT 6,
  units_per_bundle INT,
  bundle_weight_kg NUMERIC
);
```

각파이프는 KS D 3568 (일반 구조용 각형강관), 원형강관은 KS D 3566 참조. 운영 들어가면서 친구가 자주 다루는 규격부터 시드 추가.

## 9. 친구한테 확인받을 사항

다음 정보가 들어와야 시드 데이터 100% 정확해짐:

- [ ] 주거래 제강사별 번들 단위 (현대제철 / 동국 / 한국제강 등)
- [ ] 8M 외 자주 쓰는 길이가 있는지 (6M, 10M, 12M)
- [ ] 자주 다루는 강종 (SD400만? SD500도? 용접용·내진용도?)
- [ ] 폴리싱 중고 철근의 가격·중량 산정 룰 (신철근의 몇 % 차감?)
- [ ] 각파이프·강관 규격은 어떤 게 자주 나가는지 (50각, 75각 등)
